# docs

The documentation site for this repo, built on [Fumadocs](https://fumadocs.dev)
(Next.js + fumadocs-mdx). It is **not an eve agent**: like `apps/playground`,
it is a non-agent app and an accepted exception to hard rule 4
(`pnpm agent:new`) — no `agent/` directory, no evals, no eve build.

## Content model — do not duplicate docs here

The repo's canonical markdown stays where it is: `docs/*.md` (playbooks),
`README.md`, and `AGENTS.md` at the root. The pages under `content/docs/` are
thin wrappers that add frontmatter (title, description) and inline the
canonical file via fumadocs-mdx's `<include>` tag. Edit the root files to
change content; edit the wrappers only to change titles, descriptions, or
ordering (`content/docs/meta.json`).

`source.config.ts` post-processes the included markdown: it drops the
duplicate leading `# h1` (the frontmatter title already renders it) and
rewrites repo-relative links (`./testing.md`, `../AGENTS.md`, …) to the
docs pages that render those files.

Adding a new playbook = one new `docs/<name>.md` at the root, one wrapper
`content/docs/<name>.mdx`, one entry in `meta.json`, and (if it will be
cross-linked) one entry in `LINK_TARGETS` in `source.config.ts`.

## Run it

```bash
pnpm --filter docs dev     # http://localhost:3000 (redirects to /docs)
pnpm --filter docs build && pnpm --filter docs start
```

`.source/` is generated by fumadocs-mdx (postinstall and on dev/build) and is
gitignored. Search is the built-in Orama engine (`app/api/search/route.ts`) —
fully local, no keys, so `pnpm verify` stays secrets-free.

## Env vars

None. The app reads no environment variables.
